üîß Fixing Underlined Link Buttons in Bootstrap from Symphony

Problem:
Buttons styled using <span class="btn"> with an <a> inside may show underlines or incorrect styles, especially after migrating to the new CMS.

Fix:
Apply the .btn class directly to the <a> tag, like this:

‚úÖ Before:

<span class="btn btn-primary">
  <a href="mailto:x@x.com">Email Mr. X</a>
</span>

‚úÖ After (Corrected):

<a href="mailto:x@x.com" class="btn btn-primary">Email Mr. X</a>

This:

·         Removes the underline

·         Ensures full button styling

·         Improves accessibility and HTML structure

Notes:

  • The first panel (collapseOne) has:
    • class="accordion-collapse collapse show" → makes it open by default
    • aria-expanded="true" on the button → accessibility
  • Others have:
    • class="accordion-collapse collapse" → starts collapsed
    • aria-expanded="false" on the button
    • collapsed class on the buttons themselves
